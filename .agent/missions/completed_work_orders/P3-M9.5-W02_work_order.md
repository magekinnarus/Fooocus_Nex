## Objective
Extract the utility functions used by backend modules from `ldm_patched/modules/utils.py` into native backend code, eliminating all `ldm_patched.modules.utils` imports.

## Scope

### Functions to Extract

**General utilities → `backend/utils.py`:**
| Function | Used By | Purpose |
|----------|---------|---------|
| `load_torch_file()` | `loader.py` | Loads safetensors/pt files |
| `get_attr(model, key)` | `patching.py`, `weight_ops.py` | Gets nested attribute by dot-separated key |
| `set_attr_param(model, key, value)` | `patching.py` | Sets nested attribute |
| `copy_to_param(model, key, value)` | `patching.py` | Copies data into existing parameter |
| `set_attr(model, key, value)` | `patching.py` | Sets nested attribute (general) |

**LoRA key-mapping → keep in `backend/lora.py`:**
| Function | Purpose |
|----------|---------|
| `unet_to_diffusers()` | Maps ComfyUI UNet keys to diffusers format |
| `mmdit_to_diffusers()` | Maps MMDiT keys (SD3) |
| `flux_to_diffusers()` | Maps Flux keys |
| `auraflow_to_diffusers()` | Maps AuraFlow keys |
| `pixart_to_diffusers()` | Maps PixArt keys |

The key-mapping functions should be copied into `lora.py` directly (they are LoRA-specific utilities, not general utils).

## Risk: M09-W04 Regression Lessons
The previous attempt at extracting `utils` caused inference speed regression. Watch specifically for:
- **`copy_to_param` / `set_attr_param`:** These place weight tensors. Any change in device/dtype handling here will affect performance.
- **`load_torch_file`:** Uses `safetensors.torch.load_file` or `torch.load`. Ensure identical behavior including `weights_only` parameter.
- **Profile before and after:** Time 3 runs of SD1.5 generation, compare average wall-clock time.
